HTML

  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user.
  • Used to create the structure of a webpage.
  • It is made up of elements which work together to provide content and the basic framework for all pages on the web.
  • An element is a type of content on the webpage.

    • Commonly used elements are

  • Text elements
  • Image elements
  • Header elements
  • List elements

    • Common element types

  • Elements are also used to divide the content on the page to describe the content to browsers and other developers
  • Some common element types used to divide and describe a page
  • Meta-related elements
  • Content division elements
  • Semantic elements

    • Head and Body elements

  • Two elements are important to understanding how the structure works. The head and body element
  • The Head element contains information (metadata) that helps browser to render the page correctly
  • The Body element contains the content that is shown to the user

    • Metadata tags

  • title element defines the webpage. This element is extremely important for SEO
  • style element defines the CSS styles
  • base element defines the the base URL for webpage
  • link element connects an external resource to the HTML document
  • meta element defines metadata such as character set, description, keywords, author, and viewport
  • script element embeds client-side scripts in the HTML document

    • Nerd Notes

  • Elements nested inside other elements are called child elements. The element that holds all the children is the parent element
  • element and tag are used interchangeably in the web development world, but there is a difference. An element refers to the entire element from opening to colsing tag, whereas a tag refers to what appears inside the angle brackets.
  • Headers: (h1) to (h6) represent level of heading.
  • The (ul), (ol), and (li) elements represent unordered list, ordered lists, and list items. Basically represented as bulleted or numbered lists.
  • footer element appears at the bottom of the page and usually contains the author, copyright, contact, sitemap, and navigation.

CSS

  • There are 3 ways to style a webpage using CSS: inline CSS, internal CSS style sheets, and external CSS style sheet
  • Inline CSS apply to a specific HTML tag. They're useful for quick and permanent changes but are less flexible than external & internal style sheets
  • Internal CSS style sheet contains CSS rules for the webpage in the head section of the HTML file. The rules only apply to that page, and you can configure CSS classes and IDs, which are used to identify elements, to style multiple elements in the page code.
  • External CSS syle sheets are the most preffered and most common because it allows developers to keep all their CSS rules in a separate file, which makes design changes easier
  • A margin indicates how much space we want around the outside of an element
  • A padding indicated how much space we want around the content inside an element
  • Declarations contain two important components: the CSS property we want to apply and the value of the property
  • Text-align is one way to center a title or heading on a webpage
  • The img selector assigns property values to any image element in the markup
  • The display property is assigned with the block value, which assigns each image to start on a new line
  • Just because files are in the same project repository doesn't mean they know about each other. To do this, use the HTML link element.
  • A link element creates the connections to the external file
  • The href attribure specifies the location (URL) of the external resource
  • Class Attributes allow us to share a CSS rule to any element we choose by assigning the rule to a class attribute with a class selector

JavaScript

  • JavaScript is a high-level dynamic programming language hat allows users to interact with websites
  • JavaScript is read from top to bottom, and order counts (Control flow)
  • Variables are containers that store values
  • Semicolons at the end of a line indicates where a statement ends
  • JavaScript is case sensative
  • Variable Type: (String) is a sequence of text. Enclose it in single quote marks
  • Variable Type: (Number). No quotes around them
  • Variable Type: (Boolean) is a true/false value. No quotes around them
  • Variable Type: (Array) is a structure that allows you to store multiple values in a single reference
  • Variable Type: (Object) can be anything and can be stored as a variable
  • Why variables? Necessary to do anything interesting and dynamic. Think personalize messaged or change displays
  • Comments are snippets of text that can be added along with code
  • Operators are mathematical symbols that produces a result based on two values
  • Operator Type: (Addition) adds two numbers together or combines two strings. Symbol (+)
  • Operator Type: (Subtraction, Multiplication, Division) do exactly what they do in math. Symbols (-, *, /)
  • Operator Type: (Assignment) assigns a value to a variable. Symbol (=)
  • Operator Type: (Strcit equality) performs a test to see if two values are equal. It returns a true/false (Boolean) result. Symbol (===)
  • Operator Type: (Not, Does-not-equal) returns the logically opposite value of what it precedes. Example, turns true into a false. When used alongside the Equality operator, the negations operator tests whether the two values are "not" equal. Symbols (!, !==)
  • Conditionals are code structures used to test if any expression returns true or not. (if/else) statements
  • Functions are a way of packaging functionality that you wish to reuse. Good alternative to repeatedly writing the same code
  • Events are code structures that listen for activity in the browser, and run code in response
  • An array is a single variable that is used to hold a group of data
  • A function is a set of instructions that tells the computer how to perform a certain task
  • A function's name is followed by parentheses ( ).
